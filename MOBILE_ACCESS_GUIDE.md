# TaskPool 移动端访问指南

## 多种方式在手机上使用TaskPool

### 方法一：本地开发环境访问（推荐用于开发测试）

**前提条件：**
- 电脑和手机连接到同一个WiFi网络
- 电脑上运行着本地服务器

**步骤：**

1. **启动本地服务器**
   ```bash
   # 如果使用Python
   python -m http.server 8000
   
   # 如果使用Node.js
   npx serve .
   
   # 如果使用PHP
   php -S localhost:8000
   ```

2. **获取电脑的IP地址**
   - Windows: 在命令提示符中运行 `ipconfig`，找到IPv4地址（通常是192.168.x.x）
   - Mac/Linux: 在终端中运行 `ifconfig` 或 `ip addr`

3. **在手机浏览器中访问**
   - 打开手机浏览器
   - 输入地址：`http://你的电脑IP:8000`
   - 例如：`http://192.168.1.100:8000`

### 方法二：使用VS Code的端口转发功能

**步骤：**

1. 在VS Code中打开项目
2. 启动本地服务器（如Live Server插件）
3. 点击VS Code底部的"端口"部分
4. 选择"转发端口"
5. 输入项目运行的端口号（如5500）
6. 获取生成的临时链接
7. 在手机浏览器中打开该链接

### 方法三：使用ADB进行调试（Android手机）

**前提条件：**
- Android手机和USB数据线
- 已安装ADB工具
- 手机已启用开发者模式和USB调试

**步骤：**

1. **连接手机到电脑**
   - 用USB数据线连接手机和电脑
   - 确保手机已启用USB调试

2. **检查ADB连接**
   ```bash
   adb devices
   ```

3. **设置端口转发**
   ```bash
   # 假设本地服务运行在端口8000
   adb reverse tcp:8000 tcp:8000
   ```

4. **在手机上访问**
   - 打开手机浏览器
   - 输入：`http://localhost:8000`

### 方法四：部署到云服务（推荐用于长期使用）

**推荐的免费托管服务：**

1. **GitHub Pages**
   - 将项目推送到GitHub仓库
   - 在仓库设置中启用GitHub Pages
   - 几分钟后即可通过生成的URL访问

2. **Netlify**
   - 访问 https://app.netlify.com
   - 拖拽项目文件夹或连接GitHub仓库
   - 自动部署并提供免费域名

3. **Vercel**
   - 访问 https://vercel.com
   - 导入项目或连接GitHub
   - 自动部署并提供预览URL

## 移动端优化功能

TaskPool已经针对移动设备进行了全面优化：

### 响应式设计特性

- **自适应布局**：在不同屏幕尺寸下自动调整布局
- **触摸友好**：所有按钮和交互元素都适合触摸操作
- **优化的字体大小**：确保在小屏幕上清晰可读
- **手势支持**：支持滑动、点击等移动设备常用手势

### 移动端特有功能

- **添加到主屏幕**：可以将TaskPool添加到手机主屏幕，像原生应用一样使用
- **离线工作**：支持离线使用，网络恢复后自动同步
- **推送通知**：任务到期提醒（需要浏览器支持）
- **触控优化**：大按钮设计，方便手指操作

## 添加到主屏幕的方法

### Android（Chrome浏览器）

1. 打开TaskPool网站
2. 点击地址栏右侧的"更多"按钮（三个点）
3. 选择"添加到主屏幕"
4. 按照提示完成添加

### iOS（Safari浏览器）

1. 打开TaskPool网站
2. 点击底部的"分享"按钮
3. 选择"添加到主屏幕"
4. 点击"添加"按钮

## 移动端使用技巧

### 性能优化

- **保持浏览器更新**：使用最新版本的移动浏览器
- **清理缓存**：定期清理浏览器缓存以保持性能
- **关闭不必要的标签页**：减少内存占用

### 使用建议

- **定期同步**：在多设备使用时，确保定期同步数据
- **启用通知**：开启浏览器通知以接收任务提醒
- **使用手势**：熟悉滑动操作以提高效率
- **合理分类**：使用任务层级功能更好地组织任务

## 常见问题解决

### 无法访问本地服务器

**解决方案：**
1. 检查电脑和手机是否在同一WiFi网络
2. 确认防火墙没有阻止访问
3. 验证本地服务器是否正常运行
4. 尝试使用不同的端口号

### 同步功能不正常

**解决方案：**
1. 检查网络连接是否稳定
2. 确认GitHub Token和Gist ID配置正确
3. 尝试手动同步操作
4. 清除浏览器缓存后重新登录

### 界面显示异常

**解决方案：**
1. 刷新页面
2. 清除浏览器缓存
3. 更新到最新版本的浏览器
4. 检查是否支持现代CSS特性
